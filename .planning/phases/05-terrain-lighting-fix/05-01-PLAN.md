# Phase 05 Plan 01: Terrain Lighting Fix

**One-liner:** Fix poor mountain lighting by adding normal mapping, improving PBR workflow, optimizing shadows, and adjusting ambient lighting.

## Problem Statement
Current terrain lighting appears flat and unconvincing for mountains. The shader uses only basic ALBEDO with high ROUGHNESS, no normal mapping, and ambient lighting washes out directional light.

## Tasks

### Task 1: Add Normal Mapping to Terrain Shader
- Add normal map textures for grass, stone, snow, sand
- Modify terrain shader to sample and apply normal maps
- Implement triplanar normal mapping for cliffs
- Adjust normal strength based on texture type

### Task 2: Improve PBR Lighting in Terrain Shader
- Add specular highlights with appropriate intensity
- Adjust ROUGHNESS values based on material type
- Add METALLIC variation for wet/icy surfaces
- Implement proper lighting calculations with NORMALMAP

### Task 3: Optimize Directional Light and Shadows
- Increase directional light energy for better outdoor illumination
- Adjust shadow settings for large terrain (bias, normal bias, distance)
- Ensure shadows work well with normal mapping
- Test shadow quality vs performance

### Task 4: Adjust Ambient Lighting
- Reduce ambient light energy to prevent washout
- Adjust ambient color to complement directional light
- Ensure ambient works well with day/night cycle

### Task 5: Improve Day/Night Cycle Lighting
- Smooth out lighting transitions
- Ensure night lighting still shows terrain details
- Adjust sun visibility checking if causing issues

## Implementation Details

### Terrain Shader Changes:
1. Add normal map texture uniforms
2. Implement normal mapping with triplanar support
3. Add specular calculations
4. Adjust material properties per texture type

### World Scene Changes:
1. Adjust DirectionalLight3D settings
2. Update shadow parameters
3. Adjust WorldEnvironment ambient settings

### Day/Night Cycle Changes:
1. Adjust lighting transition curves
2. Ensure night ambient still shows terrain
3. Test sun visibility checking

## Success Criteria
- Mountains show clear depth and form with directional lighting
- Normal mapping adds surface detail without performance hit
- Shadows work correctly on terrain slopes
- Day/night transitions are smooth and natural
- Performance remains acceptable on Intel UHD

## Risk Assessment
- **Low risk**: Changes are to shader and lighting parameters only
- **Performance impact**: Normal mapping adds minimal overhead
- **Compatibility**: Works with GL Compatibility renderer
- **Regression risk**: Can revert individual changes if issues arise

## Testing Plan
1. Test at different times of day
2. Test on mountain peaks and valleys
3. Verify shadow quality
4. Check performance metrics
5. Ensure underwater lighting still works
